# Урок 2. Поиск и применение расценок

**Длительность:** 6 ак. ч (2 теории + 4 практики)
**Пререк:** урок 1.

## Цель урока

После урока обучающийся:
- Читает шифр расценки и понимает, к какому сборнику/отделу/таблице он относится
- Находит нужную расценку в каждой из трёх программ тремя способами (по шифру / по тексту / по дереву)
- Корректно вставляет расценку в смету с правильной единицей измерения и объёмом

## Теория (60 мин)

### 2.1. Структура сборников ГЭСН РК / ЕРЕР РК

Нормативная база организована по сборникам:

| № | Тематика |
|---|---|
| 01 | Земляные работы |
| 06 | Бетонные и железобетонные конструкции монолитные |
| 07 | Бетонные и железобетонные конструкции сборные |
| 08 | Конструкции из кирпича и блоков |
| 09 | Строительные металлические конструкции |
| 10 | Деревянные конструкции |
| 11 | Полы |
| 12 | Кровли |
| 15 | Отделочные работы |
| 16 | Трубопроводы внутренние |
| ... | ... |

> Конкретный перечень сборников и их нумерация — по актуальной СНБ РК. Уточняется методистом курса перед запуском.

Внутри сборника:
- **Отдел** (например, отдел 02 — "Кладка из кирпича")
- **Таблица** (например, таблица 08-02-001 — "Кладка стен наружных")
- **Норма** (например, норма 08-02-001-1 — "Кладка стен наружных простых, толщина 1.5 кирпича")

### 2.2. Как читать шифр расценки

Возьмём пример: **ЕРЕР 08-02-001-1**

```
ЕРЕР      — единичная расценка (если ГЭСН — то элементная норма)
08        — сборник 08 (Кладка из кирпича и блоков)
02        — отдел 02 внутри сборника
001       — таблица 001 в этом отделе
-1        — конкретная норма из таблицы (вариант: толщина, сложность и т.д.)
```

Если в шифре есть префикс региона или редакции — это уточнение, какая именно версия СНБ.

### 2.3. Три способа поиска расценки

#### Способ А. По шифру
Самый быстрый, если шифр известен (например, проектировщик уже подобрал и передал).

В каждой программе есть строка ввода шифра. Ввёл → программа открывает прямо нужную норму.

#### Способ Б. По тексту наименования работы
Используется, когда шифр неизвестен. Вводится ключевое слово ("кладка", "штукатурка", "стяжка") — программа выдаёт совпадения.

**Хитрости:**
- Чем короче ключевое слово, тем больше совпадений; чем длиннее — тем точнее
- В разных программах поиск работает по-разному (АВС-4 ищет точное вхождение, Сана может делать нечёткий поиск)
- Имеет смысл искать по корню слова: "кладк" вместо "кладка"

#### Способ В. По дереву (сборники → отделы → таблицы)
Используется, когда непонятно, в каком сборнике искать. Полезно для редких видов работ.

Обучающийся открывает сборник, идёт по содержанию, находит подходящий раздел, выбирает таблицу, в ней — норму.

### 2.4. Когда подходящей расценки нет

Распространённая ситуация. Решения по приоритету:

1. **Расценка по аналогии** — ближайшая по технологии и составу. Допустимо для простых случаев. Должно быть отражено в пояснительной записке.
2. **Корректировка состава расценки** — взять расценку и явно скорректировать состав (например, заменить материал). Программа позволяет это сделать, но нужно обоснование.
3. **Индивидуальная (фирменная) расценка** — собственный расчёт нормы по технологической карте, с нормированием трудозатрат и ресурсов. Самый трудоёмкий путь, требует утверждения.

> **Ссылка на нормативку:** допустимость каждого подхода и порядок обоснования — по СН РК 8.02-01 (раздел о применении сметных норм). Для индивидуальных расценок — отдельные методические указания КДС.

### 2.5. Единица измерения нормы

Очень частая ошибка новичков: проектный объём перепутан с единицей нормы.

Пример: норма "Кладка стен" дана на **1 м³ кладки**. У проекта — стена 5 м длиной, 3 м высотой, 0.38 м толщиной = 5.7 м³.

В смете нужно вписать **5.7** (а не 5.7 м² или 5700, как иногда пишут).

Ещё пример: норма "Окраска фасадов" — на **100 м²**. Если объём фактический 250 м², в смету идёт **2.5** (250 / 100).

**Совет:** каждый раз, вставляя позицию, сверять единицу нормы с проектным объёмом. Программа автоматически не пересчитывает — ввели "100" вместо "1" — получили завышение в 100 раз.

### Промежуточный контроль

После 4-й позиции преподаватель проверяет: расценка одна и та же во всех трёх программах? Объём пересчитан верно по единице нормы? Если есть ошибки — разбор и продолжение.

### Финальный контроль

После всех 8 позиций:
- Программы выдают одинаковую сумму "прямые затраты"? (расхождение ≤ 0.5%)
- Если нет — ищем где разошлись (типичные причины: использована другая редакция СНБ, неверная единица измерения, опечатка в объёме)
