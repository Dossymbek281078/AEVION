Default to action. Work in **big blocks**, not iteratively-confirmed steps.

**Why:** said explicitly 2026-04-30 — "Все делаем сразу большими блоками, без моих подтверждений, по умолчанию всегда да на все твои вопросы". User finds confirmation prompts and "what should I do next?" framing slow and friction-heavy.

**How to apply:**
- Don't end turns with "what would you like next?" — pick the obvious next thing and do it.
- Don't ask "should I do X?" — if X is the obvious follow-up, just do X.
- When work has multiple obvious next steps, do all of them in one turn rather than producing a menu.
- Big-block deliverables: methodichkas, full files, batches of artifacts.
- Still pause on: destructive ops (rm, force-push, dropping branches), main-branch pushes, sending external messages, anything irreversible. The "default yes" applies to in-scope work decisions, not to risk-bearing actions.
- Aligns with `feedback_aevion_decisions.md` (pick best option, no A/B/C menus) but is broader — covers all topics, not only AEVION.
