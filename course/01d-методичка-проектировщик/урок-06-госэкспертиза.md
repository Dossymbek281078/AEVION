# Урок 6. Госэкспертиза сметной документации

**Длительность:** 4 ак. ч (2 теории + 2 практики)

## Цель

После урока обучающийся:
- Знает процедуру государственной экспертизы в РК
- Готовит комплект документов под экспертизу
- Понимает типовые замечания экспертов и стратегии их избегания

## Теория (90 мин)

### 6.1. Зачем госэкспертиза

**Цели**:
1. **Контроль соответствия** проекта нормативам и стандартам РК
2. **Защита бюджетных средств** — на бюджетных объектах
3. **Безопасность объектов** — особенно повышенной опасности
4. **Унификация качества** проектирования

Экспертиза — это **барьер** перед стройкой. Без положительного заключения нельзя получить разрешение на строительство для объектов, попадающих под обязательную экспертизу.

### 6.2. Когда обязательна

В РК (по законодательству; уточнить актуальную редакцию):

#### Обязательно
- Все **бюджетные** объекты
- Объекты с **государственным участием**
- Объекты **повышенной опасности** (атомные, химические, гидротехнические)
- Объекты **особо охраняемых территорий**
- Объекты, превышающие **пороговые значения** по стоимости / площади / этажности
- Объекты культурного наследия

#### Не обязательно
- Малое жилое строительство (частные коттеджи)
- Малые ремонтно-восстановительные работы
- Некоторые типы внутренних реконструкций без затрагивания несущих

### 6.3. Кто проводит

В РК — **РГП "Госэкспертиза"** под МИИР. Имеет филиалы в регионах.

Для отдельных видов — **специальные экспертные учреждения**:
- По экологии (Минэкологии)
- По культурному наследию (Минкультуры)
- По здравоохранению (Минздрав — для медицинских объектов)
- По образованию (Минобразования — для учебных)

### 6.4. Процедура

```
1. Подача документов
        ↓
2. Регистрация и оплата (плата за экспертизу — % от стоимости)
        ↓
3. Распределение по экспертам (по разделам)
        ↓
4. Рассмотрение (срок 30-60 рабочих дней, зависит от размера)
        ↓
5. Замечания → проектировщику
        ↓
6. Доработка проектировщиком
        ↓
7. Повторная подача (если замечания существенные)
        ↓
8. ЗАКЛЮЧЕНИЕ
   ├ Положительное → можно строить
   └ Отрицательное → доработка / переделка
```

### 6.5. Что подаётся

В состав документов:
- Полный комплект проекта (АР, КР, ОВ, ВК, ЭО, СС, ПОС, **СД**)
- Пояснительные записки по каждому разделу
- Сертификаты и допуски проектной организации
- ТЭО (если требуется)
- Геологические отчёты
- Согласования (с местными властями, охранными зонами и т.д.)

Сметный раздел подаётся как **отдельный комплект** внутри общего проекта.

### 6.6. Сроки

Стандартные сроки рассмотрения (уточнить актуальные в КДС):
- Малые объекты: 30 рабочих дней
- Средние: 45 рабочих дней
- Крупные / сложные: 60 рабочих дней
- Уникальные: до 90 рабочих дней

При получении замечаний — срок **продлевается** на время доработки.

### 6.7. Стоимость

Плата за экспертизу — **% от стоимости** проектируемого объекта (по утверждённой шкале МИИР). Чем больше объект — тем больше плата (но в относительных %% — обычно меньше).

Эта плата — **не входит** в сметную стоимость СМР. Это отдельная статья ОБИН/ТЭО.

### 6.8. Типовые замечания экспертов по сметной части

#### Замечания комплектности
- Отсутствие отдельных ЛСР по разделам проекта
- Отсутствие ВОР
- Неподписанные документы
- Старая редакция СНБ или индексов

#### Методические замечания
- Неверный СН РК для типа объекта
- Неверные нормативы НР и СП
- Неверный способ применения индекса
- Неверная база начисления

#### Существенные замечания
- Двойной счёт
- Завышение объёмов
- Завышение цен (индивидуальные без обоснования)
- Применение коэффициентов без обоснования
- Использование расценок не по назначению
- Учёт работ, не предусмотренных проектом

#### Замечания по обоснованию
- Отсутствие пояснительной записки
- Отсутствие 3 КП на индивидуальные цены
- Отсутствие приказов / актов для коэффициентов

### 6.9. Стратегии избегания замечаний

#### Внутренний чек-лист перед сдачей
Пройти **полный** чек-лист (см. урок 1, раздел 1.7), не пропуская ни одного пункта.

#### Внутренняя экспертиза
Перед сдачей — попросить **другого** опытного сметчика в организации просмотреть комплект. Свежий глаз ловит то, что автор пропускает.

#### Опережающее обоснование
В пояснительной записке — **заранее ответить** на вопросы, которые задаст эксперт. Если применили нестандартный коэффициент — сразу объяснить почему. Это сокращает количество вопросов.

#### Структура документации
Грамотно структурированный комплект — эксперту удобнее, замечаний меньше.

### 6.10. Работа с замечаниями

Когда замечания получены:

1. **Прочитать все** замечания, классифицировать (формальные / методические / существенные)
2. По каждому:
   - **Снять** (если ошибка) → корректировать
   - **Защитить** (если эксперт неправ) → подготовить обоснованный ответ
3. **Связаться с экспертом** при возможности — обсудить спорные позиции до выпуска заключения
4. **Подготовить пакет ответов** + обновлённый комплект документации
5. **Сдать** на повторное рассмотрение

|---|---|
| ... | ... |

**Шаг 4:** для **5 спорных** замечаний (где эксперт мог быть неправ) — придумать **аргументированный ответ** на 0.5-1 страницу.
