# Урок 6. Знакомство с программами АВС-4, Смета РК, Сана

**Длительность:** 8 ак. ч (по 2.5 ч на каждую программу + 0.5 ч сравнение)

## Цель

После урока обучающийся:
- Узнаёт каждую из трёх программ по интерфейсу
- Открывает готовую смету в любой из них
- Находит в смете разделы / позиции / ресурсы
- НЕ умеет ещё создавать сметы — это уровень 2

## Теория (15 мин на каждую программу)

### 6.1. АВС-4

**Происхождение:** российский продукт, адаптированный для РК. Один из старейших на рынке.

**Тип:** desktop-приложение для Windows.

**Сильные стороны:**
- Огромная и стабильная нормативная база
- Привычный интерфейс для опытных сметчиков
- Богатые возможности экспорта/импорта (АРПС, Excel)
- Много печатных форм

**Слабые стороны:**
- Интерфейс выглядит устаревшим (Windows 7-стиль)
- Не работает в облаке — каждый сметчик у себя на ПК
- Совместная работа — только через файлы

**Главное окно:**
```
┌──────────────────────────────────────────────────┐
│ Меню (Файл / Правка / Документ / Расчёт / ...)   │
├──────────────────────┬───────────────────────────┤
│                      │                           │
│  ДЕРЕВО ПРОЕКТА      │   ТАБЛИЦА ПОЗИЦИЙ         │
│  ├ Объект            │                           │
│  │  ├ Раздел 1       │   Шифр | Наимен. | Объём  │
│  │  └ Раздел 2       │   ─────┼─────────┼──────  │
│  └ ССР               │   ...  | ...     | ...    │
│                      │                           │
└──────────────────────┴───────────────────────────┘
```

### 6.2. Смета РК

**Происхождение:** локальный казахстанский продукт, заточенный под СНБ РК.

**Тип:** desktop / клиент-серверная архитектура.

**Сильные стороны:**
- Нативная под нормативную базу РК
- Автоматическое обновление СНБ и индексов
- Поддержка региональных особенностей
- Поддержка от разработчика на казахском и русском языках

**Слабые стороны:**
- Меньшее распространение, чем АВС-4
- Может быть неузнаваема при переходе сметчика из других программ

**Главное окно:**
Структурно похоже на АВС-4 (дерево слева + таблица справа), но с другой раскладкой кнопок и других цветовых акцентов.

### 6.3. Сана

**Происхождение:** современная казахстанская разработка.

**Тип:** веб-приложение / облачное решение.

**Сильные стороны:**
- Работа в браузере — не нужно ставить
- Совместная работа в реальном времени
- Современный интерфейс
- Интеграции (BIM, экспорт в современные форматы)

**Слабые стороны:**
- Зависимость от интернета
- Молодая программа — могут быть пробелы в специфических случаях
- Не все опытные сметчики готовы перейти

**Главное окно:**
Веб-интерфейс, ближе к современным SaaS-приложениям. Дерево объектов, табличный редактор, возможность одновременной работы нескольких пользователей.

## Демонстрация (15 мин на каждую программу)

Преподаватель открывает в каждой программе **одну и ту же** учебную смету (на отделку комнаты — простой пример). Показывает:

1. Где меню
2. Где открывается / создаётся проект
3. Где дерево объектов
4. Где таблица позиций
5. Как раскрыть состав одной позиции (трудозатраты / ЭМ / материалы)
6. Где смотреть итоги
7. Как вызвать печать формы №1
8. Где справочники нормативной базы

## Параллельная практика (30 мин на каждую программу)

### Задание

В каждой программе обучающийся самостоятельно делает одно и то же:

1. Открывает учебный файл сметы (преподаватель раздаёт)
2. Находит раздел "Отделочные работы"
3. Находит позицию №3
4. Раскрывает её состав
5. Записывает на бумагу: шифр расценки, объём, трудозатраты, цену ЭМ, цену материалов, итого по позиции
6. Закрывает файл, открывает следующую программу — повторяет

Цель: убедиться, что **одни и те же данные** видны во всех программах в одной и той же логической структуре, просто **под разными интерфейсами**.

## Сравнение (30 мин)

После практики — общее обсуждение:

1. Какая программа показалась самой интуитивной?
2. Какая — самой сложной?
3. Где удобнее искать расценки?
4. Где удобнее видеть итоги?
5. Какую программу выбрать в реальной работе?

**Главный вывод:** программа — это **оболочка над одной и той же методикой**. Если сметчик понимает методику (это уровень 2 курса), он работает в любой программе. Выбор программы — обычно по месту работы (что у работодателя). Учиться имеет смысл во всех трёх — рано или поздно встретишь любую.

## Самопроверка

1. К какому году/тех.поколению относятся интерфейсы каждой программы?
2. Какая из трёх — облачная?
3. Какая программа считается "ветераном рынка"?
4. Какая разработана локально под РК?
5. Где находится дерево объектов в каждой программе?
6. Где смотреть итоги по смете в каждой программе?

## Домашнее задание

Установить на свой ПК АВС-4 и Смета РК (демо-версии). Сделать аккаунт в Сана. Открыть учебный файл (преподаватель пришлёт) во всех трёх и **зафиксировать минимум 3 различия в интерфейсе**, которые вы заметили.

## Ответы

1. АВС-4 — классический Windows; Смета РК — современный desktop; Сана — современный web
2. Сана
3. АВС-4
4. Смета РК и Сана — обе казахстанские; АВС-4 — российская с РК-адаптацией
5. Слева в дереве (для АВС-4 и Смета РК), сверху или сбоку (Сана)
6. В отдельной вкладке "Итоги" / в нижней панели / в шапке формы №1
