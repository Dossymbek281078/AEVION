# Урок 5. Защита сметы и оппонирование

**Длительность:** 3 ак. ч (1 теория + 2 практики)

## Цель

После урока обучающийся:
- Готовит проект к экспертизе так, чтобы минимизировать замечания
- Грамотно отвечает на полученные замечания
- Удерживает свою позицию там, где он прав, и принимает там, где не прав

## Теория (45 мин)

### 5.1. Подготовка как основа защиты

**Лучшая защита — заранее подготовленный комплект**, не дающий повода для замечаний.

Цикл подготовки:
1. **Внутренний чек-лист** (50 пунктов из урока 2) — пройти полностью перед сдачей
2. **Внутренняя экспертиза** — другой опытный сметчик проверяет ваш комплект
3. **Опережающие ответы в ПЗ** — заранее объяснить странные места
4. **Полная папка обоснований** — приложения с приказами, актами, КП
5. **Структурированная подача** — эксперту удобно искать

### 5.2. Структура пояснительной записки как инструмент защиты

В ПЗ заранее **отвечать** на типовые вопросы:

| Возможный вопрос эксперта | Где в ПЗ ответ |
|---|---|
| "Почему НР по этому нормативу?" | Раздел 4 — норматив, тип работы, ссылка на СН РК |
| "Откуда индекс?" | Раздел 5 — публикация КДС, квартал |
| "Чем обоснован коэффициент стеснённости?" | Раздел 6 — пункт ПОС + приказ + акт |
| "Откуда индивидуальная цена?" | Раздел 7 — 3 КП + выбор + обоснование |
| "Почему такой резерв?" | Раздел 8 — тип объекта, СН РК |

### 5.3. Получение замечаний

Когда замечания пришли:

#### Этап 1. Прочитать целиком
Не реагировать сразу. Прочитать **все** замечания, понять общий характер.

#### Этап 2. Классифицировать
- Формальные → быстро исправить
- Методические → разобраться, исправить или защитить
- Существенные → глубоко проверить, иногда защитить, часто исправить

#### Этап 3. Приоритизировать
- С финансовым эффектом — в первую очередь
- Влияющие на сроки — во вторую
- Формальные — в последнюю

### 5.4. Тактика снятия замечания

#### Если эксперт прав
**Не спорим.** Снимаем замечание:
1. Корректируем смету
2. Пишем в ответе: "Замечание принято. Внесена корректировка [ссылка на новую версию]. Финансовый эффект: -X тг."
3. В новой версии ПЗ помечаем изменение

**Зачем не спорить:** время — деньги. Защищать неправоту — дороже, чем исправить.

#### Если эксперт неправ
**Защищаем с обоснованием:**
1. Внимательно перечитываем замечание — точно ли неправ
2. Готовим **аргументированный ответ**:
   - Со ссылкой на пункт нормативки (с цитатой)
   - С приведением документа-обоснования (приказ, акт, КП)
   - С расчётом, если необходимо
3. Пишем **корректным тоном** — без эмоций, без обвинений
4. **Не отказываемся** от диалога — иногда стоит позвонить эксперту лично

#### Шаблон обоснованного ответа

```
Замечание №X (из заключения от ...):
[цитата замечания]

Ответ:
1. По пункту X.X СН РК 8.02-XX (ред. ...) допускается [применение, расчёт, подход].
2. В нашем случае [...]:
   - Основание 1: [ссылка на ПОС, лист, пункт]
   - Основание 2: [приказ заказчика № от]
   - Основание 3: [акт обследования № от]
3. Расчёт [...] выполнен корректно. См. приложение.

Просим оставить позицию №X в смете без изменений.

[Подпись + ФИО + должность]
```

### 5.5. Когда правота с обеих сторон

Часто бывает: эксперт прав в 50%, проектировщик прав в 50%. В этом случае:
- **Договариваемся** о компромиссном решении
- Например: "Применить коэффициент 1.10 вместо 1.15"
- Фиксируем письменно в протоколе разногласий

### 5.6. Протокол разногласий

Если по нескольким замечаниям не получается договориться — оформляется **протокол разногласий**:

```
Протокол разногласий по заключению экспертизы №...

Сторона эксперта: [...]
Сторона проектировщика: [...]

Замечание 1
- Позиция эксперта: [...]
- Позиция проектировщика: [...]
- Решение: [принято / отклонено / компромисс]

Замечание 2
- ...

Подписи: эксперт + проектировщик
```

### 5.7. Когда стоит идти на повторную экспертизу

Если первая экспертиза дала отрицательное заключение и:
- Проектировщик считает заключение необъективным
- Заказчик готов оплатить повторную
- В стране есть альтернативная экспертная организация

→ можно подать на **повторную** экспертизу. Но это:
- Дополнительные сроки (минимум +1-2 месяца)
- Дополнительные расходы
- Не гарантирует положительного заключения

### 5.8. Эмоциональный аспект

Защита смет — **стрессовый процесс**. Помнить:
- Эксперт не "враг" — у него своя работа
- Замечания — не "оскорбление" — это нормальная часть процесса
- Спокойствие и факты выигрывают, эмоции проигрывают
- Если хочется ответить эмоционально — отложить ответ на сутки, перечитать, переписать

## Практика (90 мин)

### Задание

Дано: **15 учебных замечаний эксперта** к учебному комплекту смет `[ФАЙЛ: lesson-05-zameczaniya.pdf]`.

Из них:
- 5 — эксперт **точно прав** (объективные ошибки в смете)
- 5 — эксперт **точно неправ** (некорректное замечание, должно быть снято)
- 5 — **спорные** (обе стороны имеют аргументы)

**Шаги:**

1. Классифицировать каждое замечание (правое / неправое / спорное) — обосновать.
2. По правым — написать **корректировки**: что изменить в смете и как.
3. По неправым — написать **обоснованные ответы** по шаблону из 5.4.
4. По спорным — предложить **компромиссное решение**.
5. Оформить **итоговый пакет ответов** — единый документ с разделами по замечаниям.

### Контроль

Куратор проверяет:
- Классификация совпадает с ключом куратора (с допуском в 1-2 ошибки)
- Ответы на неправые — со ссылками на нормативку и документы
- Компромиссы — разумные
