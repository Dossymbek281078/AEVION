# AEVION — local setup

Bring up a fresh dev machine to the latest `main`. The repo holds two
subprojects:

- `aevion-globus-backend/` — Node 20 + Express + PostgreSQL + vitest
- `frontend/` — Next.js 16 (App Router) + React 19

## Prerequisites

- Node.js **20.x** (or newer LTS)
- PostgreSQL **14+** running on `localhost:5432`, with a database the
  backend can connect to (default: `aevion_dev` as `postgres`)
- git
- *(optional)* GitHub CLI (`gh`) — only for opening / merging PRs from the
  command line

## Fresh clone (other laptop, first time)

```bash
git clone https://github.com/Dossymbek281078/AEVION.git aevion-backend-modules
cd aevion-backend-modules

# Install backend deps (Linux-only optional deps included for parity with CI)
(cd aevion-globus-backend && npm install --include=optional --no-audit --no-fund)

# Install frontend deps
(cd frontend && npm install --no-audit --no-fund)

# Bootstrap a local .env from the example
cp aevion-globus-backend/.env.example aevion-globus-backend/.env

# Generate strong local secrets and print them — paste into the .env above
echo "QSIGN_SECRET=$(node -e 'console.log(require(\"crypto\").randomBytes(32).toString(\"base64\"))')"
echo "SHARD_HMAC_SECRET=$(node -e 'console.log(require(\"crypto\").randomBytes(32).toString(\"base64\"))')"
echo "AUTH_JWT_SECRET=$(node -e 'console.log(require(\"crypto\").randomBytes(32).toString(\"base64\"))')"
```

Edit `aevion-globus-backend/.env` and set:

- The three `*_SECRET` values printed above (32-byte base64 each).
- `DATABASE_URL=postgresql://postgres:<password>@localhost:5432/<db_name>`

## Pull today's changes (already cloned)

```bash
cd aevion-backend-modules
git fetch origin
git checkout main
git pull origin main

(cd aevion-globus-backend && npm install --include=optional --no-audit --no-fund)
(cd frontend && npm install --no-audit --no-fund)
```

## Run dev servers

```bash
# Terminal 1 — backend (port 4001)
cd aevion-globus-backend
npm run dev

# Terminal 2 — frontend (port 3000)
cd frontend
BACKEND_PROXY_TARGET="http://127.0.0.1:4001" npx next dev
```

Open http://localhost:3000.

## Verify the stack works

```bash
# Backend tests (vitest)
cd aevion-globus-backend && npm test

# Backend type-check
cd aevion-globus-backend && npx tsc --noEmit

# Frontend type-check
cd frontend && npx tsc --noEmit

# Frontend production build (smoke)
cd frontend && BACKEND_PROXY_TARGET="http://127.0.0.1:4001" npx next build
```

## Optional: Bureau Phase A providers

The Bureau Verified-tier (`/bureau/upgrade/[certId]`) defaults to safe
in-process stub providers so end-to-end demos work offline. To wire real
KYC + payments:

```env
# KYC — Sumsub
BUREAU_KYC_PROVIDER=sumsub
SUMSUB_APP_TOKEN=<from sumsub dashboard>
SUMSUB_SECRET_KEY=<HMAC key for webhook signature>
SUMSUB_LEVEL_NAME=<applicant level configured for AEVION Bureau>

# Payment — Stripe
BUREAU_PAYMENT_PROVIDER=stripe
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_BUREAU_PRODUCT_PRICE=price_...

# Pricing override (default: 1900 cents = $19)
BUREAU_VERIFIED_PRICE_CENTS=1900
BUREAU_VERIFIED_PRICE_CURRENCY=USD
```

After the env keys are set, populate the TODO blocks in
`aevion-globus-backend/src/lib/kyc/sumsubProvider.ts` and
`aevion-globus-backend/src/lib/payment/stripeProvider.ts` (each ~30 lines
following the linked vendor docs in those files). The route layer needs
no changes.

## Common issues

- **`npm ci` fails on Linux with EUSAGE: missing @emnapi/...`**
  Lockfile drift from a Windows-only `npm install`. Fix on the Windows
  machine: `cd aevion-globus-backend && rm -rf node_modules package-lock.json && npm install --include=optional`. Commit the regenerated lockfile.

- **Backend throws `MISSING_QSIGN_SECRET`/`MISSING_SHARD_HMAC_SECRET` at
  protect time.** The required env vars are missing. Use the generator
  command above to mint local-dev values.

- **`Database "aevion_dev" does not exist`.** Create it: `createdb -U postgres aevion_dev`. Tables are auto-created on first request via `ensureTables()` — no migration step required for dev.
