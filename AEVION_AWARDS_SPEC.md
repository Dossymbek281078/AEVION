# AEVION — премии «как Грэмми / Оскар» для ИИ-контента (музыка и кино)

**Статус:** согласованная **продуктовая** модель для MVP и последующих итераций. Юридически использовать **собственные бренды Aevion** (не торговые знаки Academy Awards / GRAMMY без лицензии); смысловая аналогия — в коммуникации.

**Техническая база:** модуль **Planet Compliance** — типы артефактов `music` и `movie`, API `/api/planet`, UI `/planet` (см. `AEVION_WORKLOG_2026-03-20.md`).

**Витрины (MVP в монорепо):**

| Продукт | Маршрут | Переход в Planet |
|---------|---------|-------------------|
| Музыкальная премия | `/awards/music` | `/planet?type=music` |
| Кинопремия | `/awards/film` | `/planet?type=movie` |

На странице Planet query-параметры **`type`** или **`artifactType`** презаполняют тип, заголовок и `productKey` под трек премии.

**Публичная статистика (метрика Y и активность):** `GET /api/planet/stats` — без авторизации, JSON:

- **`eligibleParticipants`** — число пользователей с **активным** Planet CodeSymbol (`PlanetCodeSymbolHistory.validUntil IS NULL`). Основной кандидат на **Y** в формуле «проголосовало X из Y».
- **`distinctVotersAllTime`** — уникальные голосовавшие по Planet (любой артефакт).
- **`submissions`**, **`artifactVersions`**, **`certifiedArtifactVersions`** — обзорная активность.
- Опционально **`?productKeyPrefix=`** (только безопасные символы `a-z A-Z 0-9 _ -`, до 80 знаков) — поле **`scopedToProductKeyPrefix`**: заявки и версии, у которых `PlanetSubmission.productKey` начинается с префикса (для среза «только музыкальная / кинопремия»: `aevion_award_music`, `aevion_award_film`).

**Публичная карточка артефакта** `GET /api/planet/artifacts/:id/public` дополнительно отдаёт **`voteStatsByCategory`**: по каждому `categoryId` — число голосов и средняя оценка.

**Лента для витрин:** `GET /api/planet/artifacts/recent` — последние версии артефактов **с сертификатом**. Query: **`limit`** (1–25, по умолчанию 8), опционально **`productKeyPrefix`**, **`artifactType`** (`music` | `movie` | …). Используется главной страницей и витринами премий.

**Номинации (categoryId):** пресеты на публичной странице артефакта задаются в `frontend/src/lib/planetNominations.ts` (музыка / кино / code / web). Пользователь выбирает номинацию в `<select>`; один голос на пару (user, artifactVersion, category). Подписи в таблице «Голоса по номинациям» — через **`nominationLabel()`** в том же модуле.

## 1. Два отдельных сайта / приложения

| Линия | Фокус | Тип артефакта Planet |
|--------|--------|----------------------|
| **Музыкальная премия** | Треки, альбомы, звук, ИИ-аранжировки и т.д. (scope уточняется в MVP) | `music` |
| **Кинопремия** | Короткий метр, сценарии, монтаж и т.д. (scope уточняется в MVP) | `movie` |

**Требование:** пользователь воспринимает их как **два разных продукта** (разный копирайт, навигация, возможно домены или поддомены позже). **Backend и Planet — общие**, чтобы не плодить два несовместимых контура голосов и сертификатов.

---

## 2. Премии в духе Грэмми и Оскара

- **Музыкальная линия** — номинации и риторика в духе **музыкальной премии** (аналог Грэмми по подаче).
- **Кинолиния** — номинации и риторика в духе **кинопремии** (аналог Оскара по подаче).

**MVP:** достаточно **ограниченного набора категорий** (как `categoryId` в голосовании Planet) и понятных правил «одна работа — одна заявка в номинации»; расширение номинаций — после стабильного сценария.

---

## 3. Голосование и «участники Planet»

**Договорённость:** итоги и смысл голосования **связаны с аудиторией Planet**: голоса считаются **в контексте числа участников** экосистемы (участников Planet), а не как абстрактные «лайки».

### Варианты интерпретации (для реализации)

| Код | Описание | MVP |
|-----|----------|-----|
| **A** | Показ **охвата:** «проголосовало **X** из **Y** участников Planet» (Y — согласованное определение пула) | Рекомендуется как минимум для **прозрачности** |
| **B** | **Нормализация:** доля = голоса / Y (сравнимость между сезонами) | После фиксации Y и стабильной методики |
| **C** | В рейтинге — **сырые** голоса; участники Planet — **отдельная метрика** экосистемы | Возможно как упрощение первой версии UI |

**Открытый пункт (закрыть до «рабочей версии» для демо инвесторам):** кто входит в **Y** — все зарегистрированные пользователи Aevion, только пользователи с **Planet CodeSymbol**, только прошедшие онбординг Planet и т.д. Текущий код: голос привязан к `userId` (JWT); глобальный подсчёт Y — отдельная задача на backend/продукт.

## 4. Стыковка с разработкой

- **Параллельно** можно вести **две витрины** (музыка / кино), меняя в основном frontend и контент, если **контракты Planet** стабильны.
- **Один gate:** `npm run verify` из корня `aevion-core`.
- **После MVP:** улучшение **отпечатков и similarity** для `movie`/`music` — по `AEVION_27_PROJECTS_ROADMAP.md` (Planet Compliance).
