# LMS — роли участников и операционные процессы

## Роли в курсе

### Обучающийся (студент)
- Регистрируется с реальными ФИО и удостоверением личности (для сертификации)
- Проходит модули по очереди
- Сдаёт тесты и контрольные задания
- Защищает кейсы

### Куратор уровня
- Закреплён за группой (10–20 обучающихся)
- Проверяет контрольные задания (срок — 2 рабочих дня)
- Отвечает на вопросы в форумах модулей (срок — 1 рабочий день)
- Принимает кейсы уровня индивидуально
- Ведёт рейтинг группы

### Куратор-методист
- Отвечает за качество материалов
- Обновляет нормативку при изменениях
- Координирует кураторов уровней
- Анализирует статистику прохождения, выявляет проблемные модули

### Преподаватель (для очных сессий и комиссий)
- Записывает видеолекции
- Участвует в комиссиях по защите кейсов уровней 4–5
- Ведёт мастер-классы по спецтемам

### Независимый эксперт
- Привлекается для комиссии по уровню 5
- Имеет реальный опыт госэкспертизы

### Администратор LMS
- Регистрация / отчисление обучающихся
- Выдача сертификатов
- Техническая поддержка платформы

## Процесс прохождения курса (для обучающегося)

```
[Регистрация] → [Оплата] → [Доступ к Уровню 1]
       ↓
[Модуль 1.1] → [Тест] → [Контрольное] → [Проверка куратором]
       ↓                              ↓
   [< 70%]                      [Не зачтено]
       ↓                              ↓
   [Повторно]              [Доработка]
       ↓                              ↓
[Модуль 1.2] → ... (аналогично) ... → [Модуль 1.5]
       ↓
[Кейс уровня 1] → [Защита у куратора]
       ↓
   [Зачёт] → [Сертификат уровня 1] → [Доступ к Уровню 2]
       ↓
   ... (аналогично для уровней 2-5) ...
       ↓
[Итоговая аттестация] → [Комплексный сертификат курса]
```

## Процесс проверки контрольного задания (для куратора)

1. Уведомление о новой сдаче (e-mail / Slack / в LMS)
2. Открыть файл проекта в соответствующей программе
3. Пройти по чек-листу проверки (см. `03-чек-листы.md`)
4. Выставить оценку по рубрике
5. Написать комментарий (что хорошо, что нужно доработать, ссылки на материал курса)
6. Отправить ответ обучающемуся через LMS

**SLA:** 2 рабочих дня после получения сдачи.

## Процесс защиты кейса (для уровней 1–3)

1. Обучающийся выбирает дату/время защиты в календаре куратора
2. Защита проходит в видеоконференции (включена камера)
3. Длительность — 30–60 минут
4. Куратор задаёт вопросы по чек-листу защиты
5. Обучающийся даёт ответы
6. Куратор выставляет оценку и заполняет протокол защиты
7. При зачёте — открывается следующий уровень

## Процесс защиты кейса (для уровней 4–5)

1. Обучающийся подаёт заявку на защиту перед комиссией
2. Комиссия из 3 человек собирается (онлайн или очно)
3. Защита — 60–90 минут:
   - Презентация комплекта (15 мин)
   - Вопросы комиссии (45–60 мин)
   - Совещание комиссии (15 мин)
4. Комиссия выставляет оценку коллегиально
5. Протокол защиты подписывается всеми членами

## Процесс обновления нормативки

Нормативка РК — живой документ. КДС публикует:
- Ежеквартально — индексы пересчёта
- Периодически — обновления СН РК, СНБ, методических указаний

**Куратор-методист еженедельно:**
- Проверяет обновления на сайте КДС
- Обновляет ссылки и материалы курса
- Уведомляет всех обучающихся и кураторов о существенных изменениях

**При существенном обновлении нормативки** (новая редакция СН РК):
- Обновляются методички
- Перезаписываются видеолекции по затронутым модулям
- Пересматриваются кейсы (учебные параметры)
- Все обучающиеся получают сводку изменений

## Процесс аттестации преподавателей и кураторов

Раз в год — внутренняя проверка:
- Сами проходят все 5 уровней курса (обновлённую версию)
- Сдают итоговую аттестацию
- Проходят интервью с куратором-методистом
- Получают подтверждение квалификации

## Процесс работы с обратной связью

После каждого модуля — короткая анкета (3 вопроса):
1. Понятно ли изложен материал? (1–10)
2. Достаточно ли практики? (1–10)
3. Что улучшить? (текст)

Ежемесячно — методист анализирует средние оценки:
- Модули с оценкой < 7 — пересматриваются
- Жалобы на нехватку практики — добавляются упражнения
- Текстовые комментарии — учитываются в плане обновления

## KPI курса

Отслеживаются:

| KPI | Норма |
|---|---|
| Доля прошедших уровень 1 от поступивших | ≥ 90% |
| Доля прошедших уровень 5 от поступивших | ≥ 30% |
| Средняя оценка модуля | ≥ 8/10 |
| Время реакции куратора на вопрос | ≤ 1 раб. день |
| Время проверки контрольного | ≤ 2 раб. дня |
| Доля жалоб на качество проверки | < 5% |
| Удовлетворённость выпускников через 6 мес. | ≥ 8/10 |

## Финансовая сторона

Не входит в этот документ напрямую (это операционная часть бизнеса), но в LMS должны быть:
- Тарифы (отдельно по уровням, комплексные пакеты)
- Возможность оплаты от организации (для корпоративных групп)
- Учёт скидок (для повторного прохождения, для выпускников)
- Возврат при невозможности продолжать курс
