## 0. Что мы тут делаем

**Цель сессии:** AI-тренажёр сметного дела РК для учебной программы (см. `C:/Users/user/smeta-rk-kurs/`). Не конкурирует с АВС-4 / Смета РК / Сана; **обучает методике** на узком учебном корпусе.

**Не делаем:**
- Полноценную сметную программу
- Реальную интеграцию с СНБ РК (платная подписка ИСТ Эталон)
- Парсинг/импорт смет из коммерческих программ
- Регистрацию в реестре допущенных средств для бюджетных объектов

**Делаем:**
- Узкий учебный корпус: ~200 расценок, ~500 материалов, замороженный «учебный квартал» индексов
- AI-советник на типовых ошибках студента (не вычел проёмы, двойной счёт, забыл коэффициент)
- Интерфейс, **похожий концептуально** на АВС-4 / Смета РК / Сана, но не идентичный (защита от cease-and-desist)
- Все формы (1, 2, 3, КС-2, КС-3) в учебном виде
- Привязка к сквозному кейсу курса (капремонт школы №47)

## 3. Архитектура (MVP)

### Frontend
- **Next.js 14+** App Router (как остальные AEVION-приложения)
- **TypeScript** + **TailwindCSS**
- **React 18+** с `'use client'` где нужно интерактив
- Состояние локальное в компонентах + URL-параметры; глобального стора нет

### Учебный корпус
- **JSON-файлы** в `frontend/src/app/smeta-trainer/data/`
- Структура: расценки, материалы, индексы, объекты, ВОР, дефектные акты
- Валидация через **Zod** при загрузке (защита от опечаток в JSON)

### Расчётный движок
- Чистый TypeScript в `lib/calc.ts`
- Функции: `calcPosition()`, `calcSection()`, `calcLsr()`, `applyIndex()`, `applyCoefficient()`
- Без сайд-эффектов, всё функционально → легко тестировать

### AI-советник
- Использует **AEVION QCoreAI** через `aevion-globus-backend/src/routes/qcoreai.ts`
- Для MVP: 1 сценарий — обнаружение незачёта проёмов
- Промпт-инжиниринг в `lib/ai/scenarios/`

### Хранение данных
- MVP: всё в браузере (localStorage), без бэкенда
- Phase 2: Postgres через Prisma для прогресса студентов

## 5. Принципы кода

- **Не плодить абстракции «впрок».** MVP сначала.
- **Чистые функции** в расчётном движке — никаких глобальных переменных.
- **Типизация всего корпуса** через TypeScript + Zod-валидаторы.
- **Никаких длинных файлов** — если компонент > 300 строк, бить на части.
- **Defensive copy** для объектов из корпуса — состояние не должно мутировать seed.

## 6. UX-принципы

- **Узнаваемо, но не идентично.** Дерево объектов слева, таблица позиций справа — общепринято. Раскраска / шрифты / иконки — наши.
- **AI всегда видим.** Боковая панель с советами, не модалка.
- **Цвет и значение.** Зелёный — корректно, жёлтый — внимание, красный — ошибка.
- **Минимум кликов.** Студент за курс «накручивает руку» на скорость.
