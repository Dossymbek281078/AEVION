## Проблема (реальная, не маркетинговая)

ИИ генерирует миллиарды файлов в день. У автора нет способа **доказать** что это его работа — криптографически, юридически, навсегда.

Сегодня для этого нужно:
1. Нотариус или платформа регистрации — 1 сервис
2. Криптографическая подпись — 2-й сервис
3. Мониторинг плагиата — 3-й сервис
4. Механизм роялти — 4-й сервис

**Они не связаны. Данные не переходят между ними. В суде это не работает.**

---

## Решение (то что реально работает прямо сейчас)

```
register → sign → certify → vote → earn
```

Всё за один UI, одна учётная запись, один API.

### 🔐 Продукт 1: IP Trust Pipeline (QRight + QSign + Bureau)
*Целевой рынок: юридические фирмы, медиапродюсеры, разработчики ПО, AI-студии*

**Что делает:**
- **QRight** — фиксирует SHA-256 хеш с временной меткой. Доказательство «это моё, вот момент»
- **QSign** — подписывает **ML-DSA-65** (NIST FIPS 204, постквантовый стандарт, принят август 2024). Подпись останется действительной через 15 лет когда RSA сломают квантовые компьютеры
- **Bureau** — выдаёт сертификат со ссылкой на Trust Graph. Платёж через Stripe, сертификат через 10 секунд

**WOW для инвестора:**
> Harvest-now, decrypt-later. Правительства и корпорации *уже сейчас* записывают зашифрованный трафик, ожидая расшифровать через 7 лет. Документ подписанный RSA сегодня в 2031 не будет доказательством. Подписанный ML-DSA-65 — будет. Мы единственный B2C продукт с этим алгоритмом на витрине.

**Статус:** Полный E2E на проде. Stripe → cert → Trust Graph → AEC reward. Daily smoke 24/24 PASS.

## Почему сейчас? Timing

**3 конвергирующих тренда в 2026:**
1. **NIST постквантовый стандарт ML-DSA-65** — принят август 2024. Enterprise начинают требовать постквантовую подпись в контрактах. Мы уже внутри
2. **ИИ-контент-лавина** — Sora, Midjourney, Suno создают миллиарды файлов/день. Авторство стало кризисной проблемой
3. **Регуляция ИИ в ЕС (AI Act, 2025)** — требует документирование provenance для AI-generated content. Наш Bureau-сертификат это закрывает
